- 2026-06-14 **Push-metrics receiver for short-lived wrapper pods (tatara-operator#42 B1).** New `internal/pushmetrics` package: a `Receiver` that is both an HTTP push endpoint and an unchecked `prometheus.Collector`. Wrappers POST their /metrics text to `POST /internal/metrics/push?run_id=&pod=&job=` on the INTERNAL_ADDR listener (not ingress, no OIDC, mounted alongside the turn-complete callback in `turncallback.go`); the receiver parses the text (`expfmt.NewTextParser(model.UTF8Validation)` - the zero-value TextParser PANICS in common v0.67 because NameValidationScheme is unset), force-stamps run_id/pod/job identity labels on every series (so a run can't spoof another's identity), and keys the snapshot by run_id. `Collect` re-exposes all live runs on the operator's own /metrics (registered on `ctrlmetrics.Registry` in main.go) and is union-label-padded per metric name so two runs pushing the same metric with different label sets gather cleanly instead of erroring the whole scrape; `Describe` sends NOTHING (required to register as an unchecked collector that may emit arbitrary wrapper-chosen descriptors). TTL eviction (config `PUSH_METRICS_TTL`, default 5m) ages out a hard-killed pod's series at Collect time; DELETE on the same path is the wrapper's best-effort graceful cleanup. Self-metrics: `operator_pushed_runs` gauge, `operator_push_receive_total{result}`, `operator_pushed_runs_evicted_total`. Deliberately NOT the upstream Pushgateway (no expiry = silent-staleness footgun). The human approved the issue's proposed-default design with "go". REMAINING (subtasks, separate PRs): B2 wrapper-side push client in tatara-claude-code-wrapper, B3 per-repo instrumentation gaps for the long-lived services. TDD, gofmt/lint/`go test ./...` all green.
